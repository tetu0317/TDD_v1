# TC_002: ドキュメント追加テスト

## テストケース情報

| 項目 | 内容 |
|------|------|
| **テストケースID** | TC_002 |
| **テスト分類** | 単体テスト |
| **対象クラス** | DocumentSearcher |
| **対象メソッド** | `addDocument(title, path, content)` |
| **優先度** | 高 |

## テスト内容

### 事前条件（試験規格）
- DocumentSearcherインスタンスが生成済み

### 判定条件（試験合格値）
- ドキュメントオブジェクトが配列に追加され、返り値がtrueである

## 検証項目

1. 引数: `title="設計書.txt"`, `path="/docs/design.txt"`, `content="設計内容"`
2. documents配列に1件追加される
3. 追加されたオブジェクトは`{title, path, content}`の構造
4. 返り値が`true`

## テストコード例

```javascript
test('ドキュメントが正常に追加される', () => {
  const searcher = new DocumentSearcher();
  const result = searcher.addDocument(
    '設計書.txt',
    '/docs/design.txt',
    '設計内容'
  );

  expect(result).toBe(true);
  expect(searcher.documents).toHaveLength(1);
});

test('追加されたドキュメントが正しい構造を持つ', () => {
  const searcher = new DocumentSearcher();
  searcher.addDocument('設計書.txt', '/docs/design.txt', '設計内容');

  const doc = searcher.documents[0];
  expect(doc).toHaveProperty('title', '設計書.txt');
  expect(doc).toHaveProperty('path', '/docs/design.txt');
  expect(doc).toHaveProperty('content', '設計内容');
});
```

## 実装コード例

```javascript
addDocument(title, path, content) {
  const document = { title, path, content };
  this.documents.push(document);
  return true;
}
```
